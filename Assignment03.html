Here are the input/output specifications:

Problem 2

Input specification: the first line contains n. Following that are n lines of data. The i-th of these lines contains two integers, xi and yi, separated by a space, representing the x and y coordinates of the i-th police car. You may assume that n is not bigger than 10000 and that all the coordinates are integers with absolute value no greater than 10000.

Output specification: the output contains one line with the minimum possible sum of distances to every police car (notice, you will output the sum of the distances, not the best location for the donut store), followed by the end-of-line symbol "\n".

To submit your code, use the try system as follows:

try grd-665 hmwk3-1 Donut.java

or

try grd-665 hmwk3-6 donut.py

Sample input: input-1.1, input-1.2
Sample output: answer-1.1, answer-1.2



Problem 3

Input specification: the first line contains n. Following that are n lines of data. The i-th of these lines contains three numbers, separated by spaces. The three numbers represent the starting time, ending time and employer, respectively, for the i-th job. You may assume that n is no larger than 1000000. You may assume that the starting times and ending times are all positive integers no greater than 10000. You may assume that the employer is an integer with value either 0 or 1.

Output specification: the output contains a single line containing a single number: the maximum number of jobs that can be scheduled (according to the constraints of the problem), followed by the end-of-line symbol "\n".

To submit your code, use the try system as follows:

try grd-665 hmwk3-2 Jobs.java

or

try grd-665 hmwk3-7 jobs.py

Sample input: input-2.1, input-2.2
Sample output: answer-2.1, answer-2.2


Problem 4


Input specification: the first line contains n, W1, and W2. This is followed by n lines where the i-th line describes the i-th item: it contains wi and ci. You may assume that n, W1, and W2 are not bigger than 1,000 and that all the numbers fit in int.

Output specification: the output contains one line with the maximum cost of a set of items that fits in the two backpacks, followed by the end-of-line symbol "\n".

To submit your code, use the try system as follows:

try grd-665 hmwk3-3 DoubleKnapsack.java

or

try grd-665 hmwk3-8 doubleKnapsack.py

Sample input: input-3.1, input-3.2, input-3.3, input-3.4
Sample output: answer-3.1, answer-3.2, answer-3.3, answer-3.4


Problem 5

Input specification: the first line contains n and the next line contains a1, ..., an, separated by spaces. You may assume that all the numbers fit in int.

Output specification: the output contains one line with the maximum length of an increasing subsequence of a1, ..., an, followed by the end-of-line symbol "\n".

To submit your code, use the try system as follows:

try grd-665 hmwk3-4 LongestIncreasingSubseqRecursive.java
try grd-665 hmwk3-5 LongestIncreasingSubseqDP.java

or

try grd-665 hmwk3-9 longestIncreasingSubseqRecursive.py
try grd-665 hmwk3-10 longestIncreasingSubseqDP.py

Sample input: input-4.1, input-4.2, input-4.3, input-4.4, input-4.5
Sample output: answer-4.1, answer-4.2, answer-4.3, answer-4.4, answer-4.5


Note: No upper-bound on n is specified. Run both the recursive and the dynamic programming algorithm on a variety of inputs, increasing n. Provide both numerical results and analysis. It is ok if the recursive algorithm successfully runs only on the smaller test cases.
